# `shadcn-component-lock.md` — file format

This document specifies the exact contract that the generated `shadcn-component-lock.md` MUST follow. Any tool (agent, linter, CI script) that reads or writes the file should honor this format.

## Location

- **Default: inside the shadcn UI directory** — `<resolvedPaths.ui>/shadcn-component-lock.md` (e.g. `components/ui/shadcn-component-lock.md`, `src/components/ui/shadcn-component-lock.md`, or `packages/ui/src/components/shadcn-component-lock.md` in monorepos). The path is read from `npx shadcn@latest info --json`, never hardcoded.
- **Why next to the primitives:** an agent about to edit `components/ui/button.tsx` finds the lockfile in the same directory listing it sees first, so the rule is impossible to miss.
- **Pointer:** the script also maintains a `## shadcn primitives are locked` section in `AGENTS.md` at the project root, delimited by `<!-- shadcn-component-lock:pointer -->`, so agents that start reading from `AGENTS.md` are routed to the lockfile.
- **Override:** any path the developer passes via `--out`. There is exactly one lockfile per UI directory (so a monorepo with two `components/ui/` directories gets two lockfiles, each scoped to its package).

## Sections (in order)

1. **Generator banner** — HTML comment marking the file as generated.
2. **Title** — `# shadcn Component Lock`.
3. **Metadata YAML block** — fenced ` ```yaml ` block. Required keys:
   - `style` — value of `style` from `components.json` (e.g. `nova`, `vega`).
   - `base` — `radix` or `base`.
   - `tailwind` — `v3` or `v4`.
   - `iconLibrary` — e.g. `lucide`.
   - `aliases.ui` — import alias (e.g. `@/components/ui`).
   - `lockfile` — path of this file relative to the project root.
   - `generator` — `shadcn-component-lock@<semver>`.
   - `generatedAt` — ISO-8601 timestamp.
   - `source` — either `npx shadcn@latest info --json` or a fallback note.
4. **DO NOT MODIFY banner** — a blockquote addressed to coding agents. The phrase **"MUST NOT be edited in place"** must appear verbatim so simple grep-based agent rules can detect it.
5. **`## Locked components`** — a Markdown table with columns:
   `| marker | component | file | reference |`
   - `marker` is `🔒` (confirmed by CLI) or `❓` (on disk but not reported).
   - `component` is the canonical lowercase shadcn name.
   - `file` is a link to the file, **relative to this lockfile** (since the lockfile sits inside the UI directory, this is usually just the filename).
   - `reference` is a link to `https://ui.shadcn.com/docs/components/<name>`.
6. **`## How to change a primitive (the safe way)`** — fixed copy from the template.
7. **`## How this file is maintained`** — fixed copy from the template.
8. **`## Locally Modified`** *(optional)* — appears only when the generator detects a primitive whose contents diverge from upstream (via `npx shadcn@latest add <name> --diff`). Each entry includes the file path and a one-line note. Diverged files are listed here in addition to the main table; they are still locked.

## Why the format is intentionally boring

- Coding agents do not parse the file — they grep it. Keeping the headings, banner phrase, and table column order stable means a simple regex (`MUST NOT be edited in place`, or "is this file path in the lock table?") is enough to enforce the rule.
- The YAML metadata block is small enough to live in any agent's context window without crowding out actual instructions.
- A single file at the repo root means every agent finds it the same way (just like `package.json` or `tsconfig.json`).

## Example (abbreviated)

File path on disk: `components/ui/shadcn-component-lock.md`

```markdown
<!-- THIS FILE IS GENERATED BY shadcn-component-lock. DO NOT EDIT BY HAND. -->
<!-- This file lives INSIDE the shadcn UI directory (components/ui/) — every sibling file listed below is locked. -->

# shadcn Component Lock

​```yaml
style: new-york
base: radix
tailwind: v4
iconLibrary: lucide
aliases.ui: @/components/ui
lockfile: components/ui/shadcn-component-lock.md
generator: shadcn-component-lock@0.1.0
generatedAt: 2026-04-26T12:00:00.000Z
source: npx shadcn@latest info --json
​```

> 🛑 Notice to all Vibe Coding agents: …
> Every one of the 4 files listed below — sibling files of this lockfile, inside components/ui/ — MUST NOT be edited in place.

## Locked components

| | Component | File | Reference |
| - | --------- | ---- | --------- |
| 🔒 | `button` | [`button.tsx`](button.tsx) | [docs](https://ui.shadcn.com/docs/components/button) |
| 🔒 | `card`   | [`card.tsx`](card.tsx)     | [docs](https://ui.shadcn.com/docs/components/card)   |
| 🔒 | `dialog` | [`dialog.tsx`](dialog.tsx) | [docs](https://ui.shadcn.com/docs/components/dialog) |
| 🔒 | `input`  | [`input.tsx`](input.tsx)   | [docs](https://ui.shadcn.com/docs/components/input)  |
```

And at the project root, `AGENTS.md` contains:

```markdown
## shadcn primitives are locked

<!-- shadcn-component-lock:pointer -->

The files under `components/ui/` are stock [shadcn/ui](https://ui.shadcn.com/docs/components) primitives, reused across the project.
**Do NOT edit them in place.** Read [`components/ui/shadcn-component-lock.md`](components/ui/shadcn-component-lock.md) for the full list and the safe ways to change a primitive
(wrapper component → `cva` variant → `npx shadcn@latest add <component> --diff` smart-merge).

_This section is maintained by the `shadcn-component-lock` skill — re-run `node scripts/generate-lock.mjs` to refresh._
```
